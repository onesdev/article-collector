# Article Collector - 微信/知乎/小红书文章自动收藏

## Overview

本技能实现微信公众号文章、知乎文章和小红书笔记的自动采集与收藏。当接收到文章链接时，根据平台类型提取不同粒度的元数据，并将结果写入金山文档多维表进行结构化管理。

**平台提取策略**：
- **微信公众号 / 知乎**：提取完整元数据（标题、链接、摘要、关键词标签、作者、发布日期）
- **小红书**：仅提取标题和链接

**注意**：本技能的唯一产出是写入金山文档多维表的记录，不需要在本地生成任何文件（如 Markdown、HTML 等）。

## Trigger Conditions

满足以下任一条件时触发本技能：

1. 用户消息中包含 `mp.weixin.qq.com` 链接（微信公众号文章）
2. 用户消息中包含 `zhuanlan.zhihu.com` 链接（知乎专栏文章）
3. 用户消息中包含 `zhihu.com` 链接且指向文章页面（知乎回答/文章）
4. 用户消息中包含 `xiaohongshu.com` 或 `xhslink.com` 链接（小红书笔记）
5. 用户明确要求收藏/收集/保存/记录某篇文章，并提供了文章链接

## Workflow

### Step 1: Extract Article URL and Identify Platform

从用户消息中提取文章 URL，并根据域名识别平台类型：

**支持的链接格式**：

| 平台 | 链接格式 | 提取策略 |
|------|---------|---------|
| 微信公众号 | `https://mp.weixin.qq.com/s/...` 或含 `__biz` 参数的变体 | 完整提取 |
| 知乎专栏 | `https://zhuanlan.zhihu.com/p/...` | 完整提取 |
| 知乎回答 | `https://www.zhihu.com/question/.../answer/...` | 完整提取 |
| 小红书 | `https://www.xiaohongshu.com/explore/...` 或 `https://www.xiaohongshu.com/discovery/item/...` 或 `https://www.xiaohongshu.com/user/profile/...` | 仅标题+链接 |
| 小红书短链 | `https://xhslink.com/...` | 仅标题+链接 |

**平台判断逻辑**：
- URL 包含 `mp.weixin.qq.com` → 平台为 `wechat`
- URL 包含 `zhihu.com` → 平台为 `zhihu`
- URL 包含 `xiaohongshu.com` 或 `xhslink.com` → 平台为 `xiaohongshu`

如果用户消息中同时包含标题文字和链接，以链接为准，标题文字作为参考。

### Step 2: Fetch and Parse Article Content

根据 Step 1 识别的平台类型，执行不同的提取策略。

#### 微信公众号 / 知乎 — 完整提取

使用 `WebFetch` 工具访问文章 URL，提取以下信息：

- **文章标题**：页面的 `<title>` 或 `<h1>` 标签内容。微信公众号文章标题通常在 `<h1 id="activity-name">` 或 `<rich_media_title>` 中；知乎文章标题在 `<h1 class="Post-Title">` 中。
- **作者**：微信公众号作者通常在 `<span id="js_name">` 或页面的公众号名称位置；知乎作者在作者信息区域。
- **文章发布日期**：微信公众号发布时间通常在 `<em id="publish_time">` 或 `var ct` 变量中；知乎文章发布时间在 `<span class="ContentItem-time">` 中。
- **摘要**：提取文章正文前 200 字以内的核心内容作为摘要。去除 HTML 标签、特殊字符和多余空白后，精炼为 200 字以内的概述。
- **关键字标签**：根据文章全文内容，提炼 1-5 个关键字标签，使用英文逗号分隔，纯文本格式。标签应反映文章的核心主题和领域。

**WebFetch Prompt 模板（微信/知乎）**：

```
请从这篇文章中提取以下信息，以结构化格式返回：
1. 文章标题：完整的文章标题文字
2. 作者：文章作者或公众号名称
3. 文章发布日期：格式为 YYYY-MM-DD
4. 摘要：200字以内的文章核心内容概述
5. 关键字标签：1-5个反映文章主题的关键词，用英文逗号分隔

请按以下格式返回：
标题：...
作者：...
发布日期：...
摘要：...
关键字：...
```

#### 小红书 — 仅提取标题和链接

小红书笔记仅提取标题和链接，无需提取摘要、关键词、作者和发布日期。

**WebFetch Prompt 模板（小红书）**：

```
请从这篇小红书笔记中提取文章标题，以结构化格式返回：
1. 文章标题：完整的笔记标题文字

请按以下格式返回：
标题：...
```

小红书短链（`xhslink.com`）会自动重定向到完整链接，WebFetch 会跟随重定向，使用最终 URL 作为链接值。

### Step 3: Convert Date Format（仅微信/知乎）

将提取的日期（格式 YYYY-MM-DD）直接作为字符串传入多维表的 Date 类型字段。此步骤仅适用于微信公众号和知乎文章。

**重要**：KDocs 多维表的 Date 字段接受字符串格式（如 `"2026-05-16"`），而非 Unix 时间戳。时间戳格式会导致写入失败（`Internal error. Failed due to field: 文章发布日期`）。

如果日期提取失败或格式不标准，尝试用 Python 进行灵活解析：
```python
from datetime import datetime
dt = datetime.strptime("raw_date_string", "%Y-%m-%d")
formatted = dt.strftime("%Y-%m-%d")
print(formatted)
```

如果文章中无法提取到发布日期，则使用当前日期作为默认值，并在摘要末尾标注"[日期待确认]"。

**小红书无需执行此步骤**，因为不提取发布日期。

### Step 4: Write to KDocs Multi-Dimensional Table

使用 `mcp__kdocs__dbsheet.create_records` 工具将提取的数据写入金山文档多维表。

**固定参数**：
- `file_id`: `ciLdfAcM6jjU`
- `sheet_id`: `1`

#### 微信公众号 / 知乎 — 完整记录

**字段映射**：

| 提取信息     | 多维表字段名   | 值格式                          |
|-------------|--------------|---------------------------------|
| 文章标题     | 文章标题      | 纯文本字符串                     |
| 文章链接     | 链接          | 完整 URL 字符串                  |
| 摘要         | 摘要          | 200字以内的纯文本字符串           |
| 关键字标签   | 关键字标签    | 英文逗号分隔的纯文本字符串        |
| 作者         | 作者          | 纯文本字符串                     |
| 文章发布日期  | 文章发布日期  | 日期字符串，格式 YYYY-MM-DD      |

**记录创建示例**：

```json
{
  "file_id": "ciLdfAcM6jjU",
  "sheet_id": 1,
  "records": [
    {
      "fields": {
        "文章标题": "AI 技术的最新进展与未来展望",
        "链接": "https://mp.weixin.qq.com/s/xxxxx",
        "摘要": "本文探讨了2025年AI领域的几项关键技术突破...",
        "关键字标签": "AI,人工智能,深度学习,技术趋势",
        "文章发布日期": "2025-01-15",
        "作者": "科技前沿观察"
      }
    }
  ]
}
```

#### 小红书 — 仅标题和链接

小红书笔记只填充 `文章标题` 和 `链接` 两个字段，其余字段留空（不写入）。

**记录创建示例**：

```json
{
  "file_id": "ciLdfAcM6jjU",
  "sheet_id": 1,
  "records": [
    {
      "fields": {
        "文章标题": "周末探店｜这家咖啡店太出片了",
        "链接": "https://www.xiaohongshu.com/explore/xxxxx"
      }
    }
  ]
}
```

### Step 5: Confirm and Report

成功写入后，向用户确认收藏结果。

**微信/知乎文章确认格式**：

```
✅ 文章已收藏到多维表：

📝 标题：{文章标题}
🔗 链接：{文章链接}
👤 作者：{作者}
📅 发布日期：{YYYY-MM-DD}
🏷️ 标签：{关键字标签}
📋 摘要：{摘要前50字}...
```

**小红书笔记确认格式**：

```
✅ 小红书笔记已收藏到多维表：

📝 标题：{笔记标题}
🔗 链接：{笔记链接}
```

如果写入失败，向用户报告错误信息，并提供提取到的文章信息供用户手动录入。

## Error Handling

1. **文章访问失败**：如果 WebFetch 无法访问文章（如微信文章被删除、知乎文章需登录、小红书需登录），向用户说明原因，并询问是否有文章标题和摘要可以手动提供。
2. **字段提取不完整**：如果部分字段无法提取（如作者缺失），用"未知"填充缺失字段，不阻断流程。小红书无需此处理，因为仅提取标题和链接。
3. **日期解析失败**（仅微信/知乎）：使用当前日期作为默认值，并在摘要中标注"[日期待确认]"。
4. **多维表写入失败**：报告错误详情，提供已提取的文章信息供用户手动处理。

## Important Notes

- 微信公众号文章可能包含反爬机制，WebFetch 可能无法获取完整内容。此时应向用户说明，并尝试用用户提供的信息补充。
- 知乎文章部分内容可能需要登录才能查看，WebFetch 可能只获取到部分内容。
- 小红书笔记可能需要登录才能查看完整内容，WebFetch 可能仅获取到标题。如果标题也无法提取，尝试使用用户消息中附带的文字作为标题。
- 小红书短链（`xhslink.com`）会自动重定向，使用重定向后的完整 URL 作为链接值。
- 关键字标签必须是纯文本、英文逗号分隔，不使用井号或其他符号。
- 日期字段必须传入字符串格式（如 `"2025-01-15"`），不能传入 Unix 时间戳（时间戳会导致 Internal error）。
- 摘要控制在 200 字以内，突出文章核心观点和价值。
- **不要在本地生成任何文件**（如 Markdown、HTML 等），唯一产出是写入金山文档多维表的记录。
